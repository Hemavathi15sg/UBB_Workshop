# Exercise 10 - Enterprise Token Governance and Best Practices

> Goal: Define an enterprise operating model for sustainable Copilot usage.

Prefer the command line? See [Exercise CLI](exercise-cli.md).

## Context

Engineering leadership wants useful Copilot usage to grow without unmanaged spend, noisy tools, repeated prompt bloat, or inconsistent team practices.

## Step 0 - Review the Ungoverned Pattern

```text
Each team uses Copilot however they prefer. We will look at billing at the end of the quarter.
```

## Step 1 - Define the Governance Layers

| Layer | Practice |
| --- | --- |
| Individual | Prompt scorecard, context ladder, fresh chats. |
| Team | Prompt templates, instruction ownership, agent budgets. |
| Enterprise | MCP audits, dashboard, mode routing, monthly review. |

## Step 2 - Create a Team Charter

```text
Enterprise Copilot operating model:
- Monthly token-efficiency review by team.
- Default context and MCP tool policy.
- Reusable prompt catalog with owners.
- Model and mode routing guidance.
- Dashboard: usage, estimated savings, acceptance quality, overage risk.
```

## Step 3 - Build a 30-Day Backlog

| Week | Action |
| --- | --- |
| 1 | Baseline top workflows and identify waste patterns. |
| 2 | Publish instructions and prompt templates. |
| 3 | Audit MCP tools and custom agents. |
| 4 | Review dashboard and create next improvement backlog. |

## Validate

You should have a governance charter with owners, metrics, review cadence, and starter actions.

## Token Savings Analysis

Expected savings: 20-45% at team scale after adoption of core practices.
